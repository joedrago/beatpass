How it works

A BeatPass encodes not just what you type, but how you type it. The delays between keystrokes are normalized into ratios relative to the shortest gap, making the pattern tempo-independent. A seven-character password typed with the "Shave and a Haircut" rhythm carries significantly more entropy than the same seven characters alone.

Storage format: Each character is followed by a 3-digit ratio (the normalized timing × 10, zero-padded). The last character stands alone.
Example: s010h023a010v010e031!010?

Sharing: The share button copies a URL with the encoded password in the fragment (hash). Give it to a friend and see if they can type it with the right rhythm!
